Ты опытный Python/ML инженер и архитектор ML-инфраструктуры. Разработай с нуля аккуратный, расширяемый и пригодный для локального использования проект на Python для обучения и инференса моделей детекции объектов в 2D.

Цель проекта:
- обучение моделей 2D detection;
- инференс по изображениям и, при необходимости, по папке с изображениями;
- удобный локальный запуск и дальнейшее расширение проекта.

Основной стек:
- Python;
- PyTorch;
- MMDetection;
- TensorBoard;
- управление Python-зависимостями через `uv`.

Ключевые требования:
1. Создай полноценную структуру проекта, а не только набор отдельных файлов.
2. Проект должен корректно разворачиваться в локальном окружении через `uv`.
3. Создай все необходимые файлы для работы с `uv`, включая как минимум `pyproject.toml`, базовую конфигурацию проекта и инструкции по установке/запуску.
4. Отдельно добавь скрипт для установки системных и внешних зависимостей, которые не управляются `uv`.
5. Этот скрипт должен работать максимально безопасно: сначала проверять окружение, затем предлагать совместимые действия, а не слепо устанавливать всё подряд.

Требования к скрипту установки системных зависимостей:
1. Определи:
- операционную систему;
- архитектуру;
- наличие NVIDIA GPU;
- версию драйвера NVIDIA;
- доступную версию CUDA;
- наличие и версию cuDNN;
- версию Python.
2. На основе найденной информации подбери совместимые версии:
- Python;
- PyTorch;
- CUDA;
- cuDNN;
- MMDetection и связанных библиотек.
3. По возможности ориентируйся на наиболее новые стабильные версии, но только если они совместимы между собой.
4. Если автоматическое определение невозможно или информация неполная, не выдумывай значения:
- явно покажи, чего не хватает;
- предложи безопасный fallback;
- выведи понятные рекомендации пользователю.
5. Скрипт должен поддерживать как минимум режим проверки окружения (`dry-run` / `check`), чтобы можно было сначала посмотреть рекомендации без установки.

Требования к самому проекту:
1. Предусмотри понятную структуру каталогов, например:
- `src/` или пакет с исходным кодом;
- `configs/`;
- `scripts/`;
- `data/`;
- `checkpoints/`;
- `log/`;
- `outputs/` или `runs/`;
- `tests/`.
2. Реализуй базовые сценарии:
- запуск обучения;
- запуск инференса на одном изображении;
- запуск инференса на директории;
- сохранение результатов.
3. Обязательно добавь интеграцию с TensorBoard для логирования процесса обучения и, где уместно, метрик валидации.
4. Все логи TensorBoard и связанные лог-файлы должны сохраняться в папку `log/`.
5. Добавь конфигурационный слой, чтобы ключевые параметры можно было менять без правки исходников.
6. Предусмотри расширяемость под новые датасеты, модели и эксперименты.
7. Добавь базовую валидацию входных параметров и понятные сообщения об ошибках.
8. Подготовь минимальный набор тестов для критичных частей проекта.

Требования к качеству:
1. Код должен быть модульным, читаемым и логично организованным.
2. Не используй избыточную магию и скрытую логику.
3. Добавь docstring и краткие комментарии только там, где они реально помогают понять код.
4. Избегай хрупких допущений, завязанных на одну ОС или одну конфигурацию GPU.
5. Если есть несколько возможных решений, выбирай наиболее практичное для локальной разработки и дальнейшей поддержки.
